# Аудит репозитория

Каждый путь, отслеживаемый Git, проверен на то, место ли ему в публичном репозитории.

Строки объединены там, где у группы файлов одно и то же обоснование: 90 файлов исходного кода в `src/` не нуждаются каждый в отдельном абзаце «исходный код приложения, секретов нет». Всё, где оценка отличается, вынесено в собственную строку. Каждый отслеживаемый путь покрыт ровно одной строкой.

## B — опубликовано после правок

**B-1. `DECISIONS.md`, `DECISIONS.ru.md`**
Назначение: журнал инженерных решений. Публиковать: **да** — это один из самых полезных документов для читателя, который оценивает проект.
Найденный риск: в строке 12 директория проекта названа как `…\Проекты\tonwallet`. Это частичный локальный путь.
Требуемое действие: убедиться, что он ничего не раскрывает.
Что сделано: оставлено как есть. Путь уже усечён до относительного фрагмента, не несёт ни имени пользователя Windows, ни раскладки дисков, ни имени хоста, и существует он затем, чтобы объяснить реальное решение (почему структура отличается от Linux VPS, который предполагало техническое задание). Две устаревшие записи в том же файле *были* исправлены: заметка «GitHub CLI не авторизован» (владелец с тех пор вошёл) и утверждение, что ротация ключей блокирует публикацию (владелец это условие снял).

**B-2. `OPERATIONS.md`, `OPERATIONS.ru.md`, `LIMITATIONS.md`, `LIMITATIONS.ru.md`**
Назначение: руководство по эксплуатации и честные ограничения.
Найденный риск: угрозы безопасности нет, но есть **фактическая ошибка** — оба документа утверждали, что для управления пользователями нет экрана администрирования, и предлагали операторам заводить пользователей самодельным `tsx`-скриптом. Страница `/admin` существует с коммита `f9cfae3`, с обработчиками маршрутов в `src/app/api/admin/users`. Публикация этого означала бы поставку инструкции к обходному пути, который никому не нужен.
Что сделано: исправлено на обоих языках. `LIMITATIONS` теперь фиксирует настоящее ограничение — экран администрирования минимален (нет поиска, постраничного вывода и групповых действий).

**B-3. `API.md`, `API.ru.md`, `ROADMAP.md`, `ROADMAP.ru.md`, `CHANGELOG.md`, `CHANGELOG.ru.md`**
Назначение: справочник по эндпоинтам, планы, история релизов.
Найденный риск: угрозы безопасности нет; четыре фактические ошибки. `API.md` утверждал, что каждый эндпоинт используется собственными страницами приложения, но у `POST /api/analyze` нет вызывающей стороны внутри приложения — страница кошелька представляет собой серверный компонент, который вызывает `analyzeWallet()` напрямую. `API.md` не документировал ни одного эндпоинта `/api/admin/*`, хотя они существуют. `ROADMAP.md` числил уже поставленный экран администрирования в разделе «Under consideration». `CHANGELOG.md` перечислял значения происхождения как `tonapi`/`toncenter`/`demo`, опуская `mixed`, который есть в `src/domain/types.ts`.
Что сделано: всё исправлено на обоих языках. Неиспользуемая поверхность `/api/analyze` теперь прямо помечена как неиспользуемая, а не подаётся молчаливым намёком как действующая.

## D — удалено

**`docker-compose.example.yml`** — побайтово совпадает с `docker-compose.yml` за вычетом комментариев, и на него не ссылается ни один документ, скрипт или workflow. Настоящий `docker-compose.yml` публиковать безопасно: каждое значение берётся из `${VAR}`, ни один секрет не вписан внутрь. Хранить вторую копию — значит гарантированно получить расхождение между ними и не дать читателю понять, какая из них главная. **Удалён из Git.**

## Методика

1. `git ls-files` — полный отслеживаемый набор: аудит покрывает то, что реально есть в Git, а не то, что случайно оказалось в рабочем дереве.
2. `gitleaks detect` по всей истории.
3. Точечные проходы `git grep` и `git log -p` по каждому пункту таблицы находок.
4. Разбор чанков PNG, чтобы подтвердить отсутствие метаданных, а не полагаться на инструмент съёмки.
5. Чтение каждого документа рядом с кодом, который он описывает, — именно так всплыли дефекты B-2 и B-3.
