## B ‚Äî published after edits

**B-1. `DECISIONS.md`, `DECISIONS.ru.md`**
Purpose: engineering decision log. Publish: **yes** ‚Äî it is one of the more useful documents for a reader judging the project.
Risk found: line 12 refers to the project directory as `‚Ä¶\–ü—Ä–æ–µ–∫—Ç—ã\tonwallet`. This is a partial local path.
Action needed: confirm it exposes nothing.
Action taken: left as-is. It is already elided to a relative fragment, carries no Windows username, no drive layout, and no hostname, and it exists to explain a real decision (why the layout differs from the Linux VPS the brief assumed). Two stale entries in the same file *were* corrected: the "GitHub CLI is not logged in" note (the owner has since authenticated) and the claim that key rotation blocked publication (the owner waived it).

**B-2. `OPERATIONS.md`, `OPERATIONS.ru.md`, `LIMITATIONS.md`, `LIMITATIONS.ru.md`**
Purpose: operations guide and honest limitations.
Risk found: no security risk, but a **factual defect** ‚Äî both claimed user management had no admin screen and told operators to provision users with an ad-hoc `tsx` script. The `/admin` page has existed since commit `f9cfae3`, with route handlers under `src/app/api/admin/users`. Publishing this would have shipped instructions for a workaround nobody needs.
Action taken: corrected in both languages. `LIMITATIONS` now records the real limitation ‚Äî the admin screen is minimal (no search, paging, or bulk actions).

**B-3. `API.md`, `API.ru.md`, `ROADMAP.md`, `ROADMAP.ru.md`, `CHANGELOG.md`, `CHANGELOG.ru.md`**
Purpose: endpoint reference, plans, release history.
Risk found: no security risk; four factual defects. `API.md` claimed every endpoint is consumed by the app's own pages, but `POST /api/analyze` has no in-app caller ‚Äî the wallet page is a server component that calls `analyzeWallet()` directly. `API.md` documented no `/api/admin/*` endpoints despite their existing. `ROADMAP.md` listed the shipped admin screen under "Under consideration". `CHANGELOG.md` listed the provenance values as `tonapi`/`toncenter`/`demo`, omitting `mixed`, which `src/domain/types.ts` includes.
Action taken: all corrected in both languages. The unused `/api/analyze` surface is now labelled as unused rather than quietly implied to be live.

## D ‚Äî removed

**`docker-compose.example.yml`** ‚Äî byte-identical to `docker-compose.yml` apart from comments, and referenced by no document, script, or workflow. The real `docker-compose.yml` is safe to publish: every value comes from `${VAR}` and no secret is inlined. Keeping a second copy guaranteed the two would drift and gave a reader no way to know which one was authoritative. **Removed from Git.**

## Method

1. `git ls-files` for the complete tracked set ‚Äî the audit covers what Git actually has, not what the working tree happens to show.
2. `gitleaks detect` across the full history.
3. Targeted `git grep` and `git log -p` sweeps for each item in the findings table.
4. PNG chunk parsing to confirm no metadata, rather than trusting the capture tool.
5. Reading each document against the code it describes ‚Äî which is how the B-2 and B-3 defects surfaced.
